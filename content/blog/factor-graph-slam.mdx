## Factor Graph란?

Factor graph는 확률 변수 간의 관계를 표현하는 이분 그래프(bipartite graph)이다. SLAM 문제에서 factor graph는 로봇의 포즈(pose)와 랜드마크(landmark)를 변수 노드로, 센서 관측과 오도메트리 제약을 팩터 노드로 표현한다. 이를 통해 SLAM 문제를 그래프 위의 최적화 문제로 변환할 수 있다.

## MAP 추정

SLAM의 목표는 센서 관측값 $z$가 주어졌을 때 로봇의 상태 $x$ (포즈 + 랜드마크 위치)에 대한 사후 확률(posterior)을 최대화하는 것이다:

$$x^* = \arg\max_x p(x|z)$$

베이즈 정리를 적용하면:

$$x^* = \arg\max_x p(z|x) p(x)$$

Factor graph에서는 이 결합 확률을 개별 팩터들의 곱으로 분해한다:

$$p(x|z) \propto \prod_{i} f_i(x_i)$$

여기서 $f_i$는 개별 팩터(관측 모델 또는 사전 정보)이고, $x_i$는 해당 팩터에 연결된 변수들의 부분 집합이다.

## 최적화 과정

각 팩터가 가우시안 노이즈 모델을 따른다고 가정하면, MAP 추정은 비선형 최소제곱 문제로 변환된다:

$$x^* = \arg\min_x \sum_{i} \| h_i(x_i) - z_i \|^2_{\Sigma_i}$$

여기서 $h_i$는 측정 함수, $z_i$는 실제 관측값, $\Sigma_i$는 노이즈 공분산 행렬이다. 이 문제는 Gauss-Newton 또는 Levenberg-Marquardt 알고리즘을 사용하여 반복적으로 풀 수 있다. 각 반복에서 선형화된 시스템 $H \delta x = b$를 풀어 상태를 업데이트한다.

## GTSAM을 활용한 구현

Google에서 개발한 GTSAM 라이브러리를 사용하면 factor graph 기반 SLAM을 쉽게 구현할 수 있다. 다음은 2D pose graph SLAM의 간단한 예시이다:

```python
import numpy as np
import gtsam

# Factor graph 생성
graph = gtsam.NonlinearFactorGraph()

# 노이즈 모델 정의
prior_noise = gtsam.noiseModel.Diagonal.Sigmas(np.array([0.3, 0.3, 0.1]))
odometry_noise = gtsam.noiseModel.Diagonal.Sigmas(np.array([0.2, 0.2, 0.1]))
loop_noise = gtsam.noiseModel.Diagonal.Sigmas(np.array([0.1, 0.1, 0.05]))

# 초기 포즈에 prior factor 추가
graph.add(gtsam.PriorFactorPose2(
    1, gtsam.Pose2(0.0, 0.0, 0.0), prior_noise
))

# 오도메트리 factor 추가
graph.add(gtsam.BetweenFactorPose2(
    1, 2, gtsam.Pose2(2.0, 0.0, 0.0), odometry_noise
))
graph.add(gtsam.BetweenFactorPose2(
    2, 3, gtsam.Pose2(2.0, 0.0, np.pi / 2), odometry_noise
))
graph.add(gtsam.BetweenFactorPose2(
    3, 4, gtsam.Pose2(2.0, 0.0, np.pi / 2), odometry_noise
))
graph.add(gtsam.BetweenFactorPose2(
    4, 5, gtsam.Pose2(2.0, 0.0, np.pi / 2), odometry_noise
))

# 루프 클로저 factor (노드 5 -> 노드 1)
graph.add(gtsam.BetweenFactorPose2(
    5, 1, gtsam.Pose2(2.0, 0.0, np.pi / 2), loop_noise
))

# 초기값 설정
initial = gtsam.Values()
initial.insert(1, gtsam.Pose2(0.0, 0.0, 0.0))
initial.insert(2, gtsam.Pose2(2.3, 0.1, -0.2))
initial.insert(3, gtsam.Pose2(4.1, 0.1, np.pi / 2))
initial.insert(4, gtsam.Pose2(4.0, 2.0, np.pi))
initial.insert(5, gtsam.Pose2(2.1, 2.1, -np.pi / 2))

# Levenberg-Marquardt 최적화 수행
params = gtsam.LevenbergMarquardtParams()
optimizer = gtsam.LevenbergMarquardtOptimizer(graph, initial, params)
result = optimizer.optimize()

# 최적화 결과 출력
for i in range(1, 6):
    pose = result.atPose2(i)
    print(f"Pose {i}: x={pose.x():.3f}, y={pose.y():.3f}, theta={pose.theta():.3f}")
```

## iSAM2: 증분적 최적화

실시간 SLAM에서는 매번 전체 그래프를 최적화하는 것이 비효율적이다. iSAM2(Incremental Smoothing and Mapping)는 새로운 팩터가 추가될 때 영향을 받는 변수만 부분적으로 업데이트하는 증분적 최적화 알고리즘이다. Bayes Tree 자료구조를 활용하여 변경된 부분만 재계산하므로, 대규모 SLAM 문제에서도 실시간 처리가 가능하다.

## 마무리

Factor graph 기반 SLAM은 문제의 구조를 명확하게 표현하고, 다양한 센서와 제약 조건을 유연하게 통합할 수 있는 프레임워크이다. GTSAM과 같은 라이브러리를 활용하면 비교적 간단하게 구현할 수 있으며, iSAM2를 통해 실시간 응용도 가능하다.
